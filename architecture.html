Hybrid model routing

Not every task deserves a frontier model. JellyClaw routes by stakes, not habit.

☁️

Claude

Routed to high-stakes work: architectural decisions, code review, anything touching production, and any Tier 2 approval reasoning. Costs tokens, buys judgment.

State & audit log

Nothing happens off the record.

🗄️

SQLite state store

Every agent, task, and message lives in a single local SQLite file. No external database to run, back up, or secure — copy one file and you have the whole system's memory.

📜

Full audit log

Every decision, tool call, and model response is logged with a timestamp and the agent that made it. Replay any task end-to-end from the dashboard.

Human-in-the-loop approval tiers

You decide how much autonomy each type of action gets.

TIER 0

Auto-approve

Low-risk, reversible actions — reading files, drafting text, running read-only queries. Agents execute immediately, no wait.

TIER 1

Batched digest

Medium-risk actions proceed automatically and land in your next Telegram digest, so you can review them in one pass and intervene if something looks off.

TIER 2

Must approve

High-risk actions — deploys, payments, external messages, deletions — pause and wait for your explicit yes/no in Telegram before running.

🛑

The kill switch

One command — jellyclaw stop --all or a single Telegram message — halts every agent immediately. No queued task executes, no in-flight tool call is retried. Full stop, always available.

the stop file — works even if the CLI is wedged 
touch ~/.jellyclaw/STOP  # every agent halts on its next tick
