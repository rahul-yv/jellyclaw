Quick Start

JellyClaw is a single install script plus a config file. If you already have Ollama running, you can have your first agent working in under ten minutes.

terminal 
curl -fsSL https://get.jellyclaw.dev | bash
cd jellyclaw && ./setup.sh

That's it for the engine. The rest of this page covers requirements, the Telegram bot, and configuring your first agent.

Requirements

RequirementMinimum
RAM16GB (for local Ollama models)
OSLinux or macOS
OllamaInstalled and running locally
Claude API keyOptional, for high-stakes tasks
Telegram accountFor reports and approvals

JellyClaw runs entirely on your own hardware. There's no hosted backend and no required third-party account — Claude and Telegram are both optional but recommended.

Installation

The install script pulls the JellyClaw binary, sets up a local SQLite database for state and audit logs, and drops a starter .env file.

terminal 
# download and run the installer
curl -fsSL https://get.jellyclaw.dev | bash

# or clone and build from source
git clone https://github.com/rahul-yv/jellyclaw.git
cd jellyclaw
./install.sh

Once installed, verify it with jellyclaw --version. If Ollama isn't running yet, start it with ollama serve before continuing.

Telegram Bot Setup

JellyClaw sends reports and approval requests through a Telegram bot that you own. Setup takes about two minutes — the whole exchange with @BotFather looks like this:

You/newbot
BotFatherAlright, a new bot. How are we going to call it? Please choose a name for your bot.
YouJellyClaw Reports
BotFatherGood. Now let's choose a username for your bot. It must end in bot.
Youjellyclaw_reports_bot
BotFatherDone! Use this token to access the HTTP API:
1234567890:ABCxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Keep your token secure and store it safely.
  1. Copy the token from BotFather's last message.
  2. Add it to your .env as TELEGRAM_BOT_TOKEN, then restart JellyClaw.
  3. Message your bot once from your own Telegram account. JellyClaw auto-detects your chat_id from that first message — no manual lookup needed.
.env 
TELEGRAM_BOT_TOKEN=1234567890:ABCxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
# TELEGRAM_CHAT_ID is filled in automatically after your first message

From here on, your morning digests, approval requests, and kill-switch commands all run through this bot.

Configuration

All runtime settings live in .env and jellyclaw.yml. The essentials:

jellyclaw.yml 
brain:
  high_stakes: claude       # reasoning, code review, approvals
  routine: qwen2.5:14b      # local Ollama model for everyday tasks

approvals:
  default_tier: 1           # 0 = auto, 1 = notify, 2 = must approve

schedule:
  timezone: America/Los_Angeles

Anthropic API keys go in .env as ANTHROPIC_API_KEY. If it's absent, JellyClaw runs entirely on local Ollama models.

First Agent

Agents are defined with a short YAML file and started with the CLI.

terminal 
jellyclaw agent create --name "research-assistant" --role worker
jellyclaw agent start research-assistant
jellyclaw task assign research-assistant "Summarize competitor pricing pages weekly"

Check progress any time with jellyclaw dashboard (opens the live dashboard on localhost:4200), or just wait for the Telegram report.

Troubleshooting

The four issues that cover most first-run problems.

ollama: command not found

JellyClaw needs Ollama installed and on your PATH. Install it, then start the server before launching JellyClaw.

terminal 
curl -fsSL https://ollama.com/install.sh | sh
ollama serve
Port 5000 already in use

On macOS, AirPlay Receiver commonly claims port 5000. Either turn it off in System Settings, or move JellyClaw's API to another port in .env:

terminal 
# see what's holding the port
lsof -i :5000
# then set a new one in .env
JELLYCLAW_PORT=5050
Telegram bot not responding

Three checks, in order: the token in .env matches what BotFather gave you exactly; you've sent the bot at least one message from your own account (JellyClaw can't message you first); and JellyClaw was restarted after the token was added. If all three pass, check the logs:

terminal 
jellyclaw logs telegram --tail 50
Agent stuck in a loop — how to kill switch

Halt everything immediately with the CLI, a Telegram message to your bot (/stop), or the stop file — whichever you can reach fastest. No queued task executes after the stop.

terminal 
jellyclaw stop --all
# or, if the CLI itself is wedged:
touch ~/.jellyclaw/STOP

FAQ

Does JellyClaw send my data anywhere?

Only where you tell it to. Local models never leave your machine. Claude calls go to Anthropic's API only when a task is routed to high_stakes.

Do I need a Claude API key?

No. JellyClaw runs fully on local Ollama models without one. A Claude key improves quality on complex or high-stakes work.

What happens if an agent tries something risky?

Every action is tagged with an approval tier. Tier 2 actions pause and wait for your explicit approval in Telegram before executing.

Can I run this on a Raspberry Pi?

Not recommended — local model inference needs 16GB+ RAM. A small VPS or an old desktop works well.

Is this really free?

Yes. JellyClaw is MIT licensed and free forever for self-hosted use. See the license page for details.
