The hybrid brain: Claude + local models

Every agent platform eventually answers one question: which model runs the show? The fashionable answers are the extremes. Route everything to a frontier model and you get the best judgment money can buy — billed per token, all night long. Route everything to local models and you get privacy and a zero marginal cost, plus an agent that occasionally reviews your architecture the way a well-meaning intern would.

JellyClaw's answer is deliberately unfashionable: both, routed by stakes.

The insight is that overnight agent work is wildly lopsided. Watch a night's audit log and most entries are routine: summarize this page, classify this issue, draft this test, rename this variable across four files. A local qwen model on a 16GB machine handles these fine — not impressively, fine — and "fine, free, and private" beats "brilliant and metered" for work whose failure mode is a slightly clumsy sentence in a digest.

But a thin slice of the night is different in kind, not degree. Reviewing a diff that touches payments. Deciding whether a migration is safe to run. Reasoning about whether an action deserves Tier 2 human approval at all. These tasks share two properties: getting them wrong is expensive, and getting them right requires judgment — the thing frontier models are genuinely better at. That slice goes to Claude. It might be five percent of the calls and ninety-five percent of the value at risk.

So the router asks one question per task: what happens if this goes wrong? Reversible-and-cheap goes local. Expensive-or-irreversible goes to Claude — and often also to a human, because the model tier and the approval tier are cousins. It's the same triage a good engineering manager does with a mixed-seniority team, which is exactly what a hybrid brain is.

The economics land where you'd hope. A night of purely-frontier agent work costs real money — fine for a funded team, corrosive for an indie hacker deciding whether to leave the agents on. In the hybrid setup, the local model absorbs the bulk of the volume, and the Claude bill tracks the number of consequential decisions, not the number of tasks. You end up paying for judgment, which is the only thing worth paying for.

Privacy lands well too. Routine tasks — the ones most likely to touch your codebase broadly — never leave your machine. What reaches Anthropic's API is the narrow, deliberate slice you configured, and the audit log shows every one of those calls. If you set no API key at all, JellyClaw simply runs everything locally: worse judgment on hard calls, zero external dependencies. That degradation path being graceful is a feature we designed for, not an accident.

There's an honest cost to hybrid: two model behaviors to understand instead of one, and a routing config you can get wrong. We mitigate it by defaulting conservatively — when a task's stakes are ambiguous, it routes up, to the better model and the stricter approval tier. You can tighten the budget once you trust the system; you shouldn't have to trust it on day one.

The one-model story is cleaner to pitch. The two-model story is cheaper to run, safer where it counts, and private where it matters. We'll take pragmatic over clean every time — that's the hybrid brain.
